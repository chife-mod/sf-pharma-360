## Tier 1 — обе модели согласны
- 💎 Сильная визуальная идентичность (Market360 DNA, не generic-темплейт).
- 💎 Богатая, хорошо упакованная карточка DOL; tier-звёзды-рампа считываются мгновенно.
- ⚠️ KPI-hero занимает слишком много экрана для ежедневного инструмента → схлопывать при скролле.
- ⚠️ Мало продуктовой глубины: нет bulk actions / multi-select / compare / saved views / export / "add to campaign". Главная общая претензия — пока «витрина», а не «инструмент решения».

## Tier 2 — расхождения (вердикт оркестратора)
- **Декор/glow:** Codex — убрать 15–25%, больше pharma-credibility; Gemini — «безупречно для питча». → Для питча драма уместна; для зашипленного продукта Codex прав (фарма консервативна).
- **Иерархия карточки:** Codex — всё равноважно, нет primary-score; Gemini — IA сила. → Сделано хорошо, но иерархия плоская; стоит выделить один primary-метрик.
- **Мобайл:** Codex — «ужатый десктоп». Клиппинг уже починен (375/390), но критика плотности справедлива.

## Tier 3 — что обе недооценили (синтез)
- Это **один экран**, а senior-портфолио = **case study** (problem → process → decisions → outcome). Живой «до»-портал — готовый *before*.
- Нет диапазона состояний: hover / empty / loading / error + страница одного DOL.

## Roadmap к 9–10
1. ✅ **Hero компактнее + ручной minimise** — *сделано 2026-06-03*. Hero ~30% ниже по умолчанию (убрана подпись-subcopy, KPI-плитки 168→116px, padding 32→24). Кнопка −/+ в углу сворачивает плашку в тонкую полоску только с заголовком (всё поднимается). Scroll-коллапс откатили — он давал layout-jump (прыжок контента при пересечении порога); ручной toggle меняет высоту только по клику = без прыжка.
2. ⬜ Каталог → **decision-workflow**: multi-select + compare + add-to-campaign + один primary-score в карточке.
3. ⬜ **Detail-страница DOL** + состояния (hover/empty/loading/error).
4. ⬜ Enterprise-density режим: table/list toggle, compact cards, saved filters, pinned metrics.
5. ⬜ Снизить декоративность на ~15–25% для «продакшн»-версии (питч-версию оставить яркой).
6. ⬜ Обрамить **case study'ем** (before/after, процесс, решения).
